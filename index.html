Getting started

First of all you have to add some JavaScript to your page. It's a rule of thumb to include scripts at the bottom of the page - near closing BODY element. As rCarousel is jQuery UI widget you have to include not only jQuery library but jQuery UI one as well.

<script type="text/javascript" src="your_path/rcarousel/lib/jquery-1.4.2.min.js"></script>
<script type="text/javascript" src="your_path/rcarousel/lib/jquery.ui.core.js"></script>
<script type="text/javascript" src="your_path/rcarousel/lib/jquery.ui.widget.js"></script>
<script type="text/javascript" src="your_path/rcarousel/lib/jquery.ui.rcarousel.min.js">
</script>

You also have to add some CSS styles used by the plugin - add this line in the HEAD section:

<link type="text/css" rel="stylesheet" href="your_path/rcarousel/css/rcarousel.css" />

When you've finished with it you can place your carousel. But how? There are two kinds of carousels: a hardcoded and a dynamic one. Let's cover the former first.

I'll show you what structure looks like:

<div class="rcarousel">
	<div class="wrapper">
		<ul>
			<li><img src="001.jpg" /></li>
			<li><img src="002.jpg" /></li>
			<li><img src="003.jpg" /></li>
		</ul>
	</div>
</div>

So the hardcoded carousel is when you create a structure manually. Every carousel image is placed inside LI element. DIV with class wrapper is crucial and cannot be omitted. No one element is allowed to be placed inside DIV.wrapper but UL (with its LI elements). If you do an error will occur. Containing DIV for DIV.wrapper should have class .rcarousel to style carousel properly.

You can also use a dynamic carousel. In this case, just place this code in your page:

<div class="rcarousel"></div>

Having placed your HTML code, you can run rCarousel plugin.
This:

<script type="text/javascript">
	jQuery(function($) {
		$("#carousel").rcarousel();
	});
</script>

will place simplest carousel on the page. It's worth noting that carousel will use default options and no navigation.

Navigation

Hey! My carousel is useless and I can't control it! Take it easy! It's nothing to worry about. Developing rCarousel I decided that a user should take care of his or her carousel's skin.I'm aware that rCarousel is quite versatile and that there is no point in designing custom skins because it's trivial to write own. Firstly, you don't have to use navigation at all! I showed it in the simple carousel example but present it again here:

jQuery(function($) {
	$("#carousel").rcarousel({
		auto: {enabled: true}
	});
});

As you see, carousel makes use of auto option. enable: true just enables automatic scrolling, so navigation is redundant. But if you'd like to add some arrows, buttons, etc. for next and previous action you can do it too! Let me show you:

<div class="rcarousel">
	<div class="wrapper">
		<ul>
			<li><img src="images/001.jpg" /></li>
			<li><img src="images/002.jpg" /></li>
			<li><img src="images/003.jpg" /></li>
		</ul>
	</div>
	<a href="#" id="ui-rcarousel-next"><img src="images/go-next.png" /></a>
	<a href="#" id="ui-rcarousel-prev"><img src="images/go-previous.png" /></a>
</div>

Pay special attention on anchor elements (A). They use special identifiers:

  • ui-rcarousel-next
  • ui-rcarousel-prev

By default rCarousel looks for these ids and attach to them click handler. This means that clicking on the a#ui-rcarousel-next will trigger show next elements action. But it doesn't have to be an A element! It can be whatever valid HTML element. It also can be called what you like, simply use navigation option:

navigation: {
  prev: ".someFancyClass",
  next: "#andFunnyID"
}

It's important that both prev and next assume that you use valid jQuery representaion of class and id. To recap:

  • ".thisIsAClass"
  • "#thisIsAnID"

Defining width and height of elements

All elements have the same width and height. By default it is 100x100 px. You can change it with width and height option. There is no need to mess around in CSS like in other carousels. Please keep in mind that elements which exceed default size will be trimmed to the default one.

defining number of visible elements and step

visible option is responsible for number of elements that you see on a page. By default there are 3 elements visible. As for visible, step is for number of new elements that will be loaded next. For example you can have 5 elements visible in your carousel and load 3 new each time you click on next or previous buttons. For step following formula applies:

0 < step <= visible

So in above example with 5 elements visible you can only use following values for step: 1, 2, 3, 4, 5

changing options at a run-time

Some options can be changed at a run-time, that is after rCarousel initialisation.
These options are:

  • step
  • speed
  • navigation
  • auto

After you've run carousel you can set new values for these options this way:

$("#carousel").rcarousel("option", "auto", {enabled: false});

In example above you disable automatic scrolling.
Some more examples:

$("#carousel").rcarousel("option", "step", 1);
$("#carousel").rcarousel("option", "speed", 300);
$("#carousel").rcarousel("option", "navigation", {prev: ".previous"});

Perhaps you ask yourself why we can't change width, height, visible or orientation. It's simple. In some way they are destructive because they could broke your nifty layout. Once you set width, height, visible and orientation there is no way to change it in a run-time.
It is worth noting that after change of 'step' carousel is re-drawn due to internal pages generation.

Loading elements on demand

As in changing options at a run-time section you can also load elements after plugins' initialisation. Because rCarousel's API exposes load method you can load elements as in below example:

$("#carousel").rcarousel("load", {path: "path_to_json.json", format: "json"});

or

$("#carousel").rcarousel("load", {path: "path_to_xml.xml", format: "xml"});

Structure of valid JSON and XML files

JSON

{
	"paths": [
		"images/001.jpg",
		"images/002.jpg",
		"images/003.jpg",
		"images/004.jpg",
		"images/005.jpg",
	]
}

XML


<paths>
	<path>images/001.jpg</path>
	<path>images/002.jpg</path>
	<path>images/003.jpg</path>
	<path>images/004.jpg</path>
	<path>images/005.jpg</path>
</paths>

Configuration options

Option Type Default value Description
visible integer 3 Number of visible elements
step integer 3 Number of elements to scroll by
width integer 100 Width of carousel's elements
height integer 100 Height of carousel's elements
speed integer 1000 Speed in milliseconds of scrolling animation
margin integer 0 Margin between elements
orientation string ["horizontal" | "vertical"] "horizontal" Orientation of carousel
auto object {enabled: boolean, direction: string ["next" | "prev"], interval: integer {enabled: false, direction: "next", interval: 5000} Enables or disables automatic scrolling.
startAtPage integer 0 Which page load first
navigation object {next: string, prev: string} {next: "#ui-rcarousel-next", prev: "#ui-rcarousel-prev"} Attach click handler to given class ".class" or id "#id"
remote object {path: string, format: string ["json" | "xml"]} no defaults Load elements from given path (URI)
start event no default Triggered when carousel is ready to use
pageLoaded event no default Triggered when page is loaded (scrolled into view)

Public API

Method Usage example Description
load $("#carousel").rcarousel("load", {path: "path_to_json.json", format: "json"}); Loads elements at a run-time and on demand
next $("#carousel").rcarousel("next"); load next elements by step - elements are scrolled left
prev $("#carousel").rcarousel("prev"); load previous elements by step - elements are scrolled right
goToPage $("#carousel").rcarousel("goToPage", 5); goes to given page
getTotalPages $("#carousel").rcarousel("getTotalPages"); returns number of total pages
